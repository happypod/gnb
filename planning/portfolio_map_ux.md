# GNB Planning 웹사이트 - Portfolio 지도 UX 설계 (v1.0)

본 문서는 GNB Planning 웹사이트의 핵심 검증 장치인 Portfolio 페이지의 UX 구조, 지도-카드 연동 로직, 데이터 스키마를 정의합니다. "나열"이 아닌 "검증"에 초점을 맞춥니다.

---

## 📄 A. Portfolio 페이지 UX 구조 설명서

### 1. 페이지 목적

- **검증:** 단순 실적 리스트가 아니라, "전국 단위 수행 능력"과 "유사 사업 성공 경험"을 시각적으로 즉시 증명합니다.
- **탐색:** 공무원이 본인의 관심 지역/사업 유형을 1분 내에 찾아보고, 상세 정보를 통해 GNB의 역량을 확신하게 합니다.

### 2. 주요 구성 요소 (Layout Strategy)

- **Filter (좌측/상단):** 원하는 조건을 빠르게 좁히는 컨트롤 타워. 직관적인 버튼/드롭다운 형태.
- **Map (중앙 - Base Layer):** 대한민국 전도가 기본. 줌인/아웃 가능하며, 지역별 프로젝트 밀집도를 핀(Pin) 또는 클러스터(Cluster)로 표현.
  - _PC:_ 화면 중앙을 넓게 차지하여 규모감 강조.
  - _Mobile:_ 상단에 지도가 위치하고 하단에 카드가 리스팅됨.
- **List (우측/하단 - Floating/Drawer):** 지도와 연동되어 현재 보고 있는 영역(Viewport) 내의 프로젝트만 리스팅하거나 필터링된 결과를 카드 형태로 보여줌.

### 3. 사용자 흐름 요약 (User Flow)

1.  **진입:** 페이지 로딩 시, 전국 지도에 GNB가 수행한 프로젝트 핀이 꽉 차 있어 규모감(압도감)을 줌.
2.  **탐색 (Filter/Map Interaction):**
    - _Scenario A (지역 기반):_ 지도에서 본인의 지역(예: 전라도)을 줌인하거나 클릭 -> 해당 지역 프로젝트만 리스트업.
    - _Scenario B (사업 기반):_ 필터에서 '어촌뉴딜' 선택 -> 관련 없는 핀은 흐려지고 해당 핀만 강조됨.
3.  **확인 (Card selection):** 리스트 또는 핀을 클릭하여 요약 카드 확인 (사업명, 성과).
4.  **전환 (Detail View):** '상세 보기' 클릭 시 상세 페이지로 이동하여 문제 해결 과정 정독.

## 📄 B. 지도–카드–필터 연동 다이어그램

### Interaction Logic

**"지도에서 찾고, 카드로 이해하고, 상세로 확신한다."**

```mermaid
graph TD
    A[사용자 진입] --> B{탐색 방식 선택}

    B -->|지도 조작| C(지도 줌인/이동)
    B -->|필터 선택| D(사업유형/지역/연도 설정)

    C --> E[Map Viewport 변경]
    D --> E

    E --> F{조건에 맞는 데이터 추출}

    F --> G[지도: 핀 재배치/하이라이트]
    F --> H[리스트: 관련 프로젝트 카드 정렬]

    G -->|핀 클릭| I[해당 프로젝트 카드 포커스 & 툴팁]
    H -->|카드 클릭| J[지도 내 해당 지역으로 이동 & 핀 활성화]

    I --> K[상세 정보 보기 (클릭)]
    J --> K

    K --> L[Case Study 상세 페이지 이동]
```

### 상호 반응 상세 (Sync)

- **필터 -> 지도/리스트:** 필터 선택 시 지도 위의 핀이 즉시 필터링(숨김/표시) 되고, 리스트도 동일하게 갱신됨.
- **지도 이동 -> 리스트:** 지도를 확대하여 특정 시/군만 보고 있다면, 리스트도 "현재 지도 영역 내 프로젝트" 우선 보기 옵션을 제공 (또는 전체 리스트 유지하되 해당 지역 아이템 하이라이트).
- **카드 호버 -> 지도:** 리스트의 카드에 마우스를 올리면, 지도상의 해당 위치 핀이 튀어오르거나 색상이 변함 (위치 인지 강화).

## 📄 C. 필터·카드 정보 항목 정의표

### 1. 필터 항목 (Search Criteria)

| 구분            | 항목명                                           | 필수 여부 | 이유 및 용도                                                            |
| :-------------- | :----------------------------------------------- | :-------- | :---------------------------------------------------------------------- |
| **사업 유형**   | `어촌` `농촌` `지역개발` `연구/정책`             | **필수**  | 발주처가 가장 먼저 구분하는 기준. (예: 어촌계 주무관은 농촌 사업 안 봄) |
| **지역**        | `전국` `강원` `경기` `충청` `전라` `경상` `제주` | **필수**  | "우리 지역 일을 해봤나?" 검증용. 광역 단위 그룹핑 권장.                 |
| **연도**        | `최근 3년` `전체`                                | 선택      | 최신 수행 실적을 우선 보여주거나, 회사의 오랜 업력을 보여주는 용도.     |
| **키워드 태그** | `#관광` `#주민역량` `#시설조성` `#기본계획`      | 선택      | 세부 과업 내용으로 검색 니즈 충족. (예: '관광' 특화 사업 찾기)          |

### 2. 프로젝트 핀 (Map Pin) 정보

- **표시 정보:** 아이콘(사업 유형별 컬러 구분), 지역명(간략, 예: 부안)
- **Click/Hover 시 툴팁:** 사업명, 수행 연도

### 3. 프로젝트 카드 (List Card) 정보

| 정보 항목         | 데이터 예시                            | 제거 불가 | 이유                                                         |
| :---------------- | :------------------------------------- | :-------- | :----------------------------------------------------------- |
| **Thumbnail**     | 현장 사진 / 조감도                     | O         | 텍스트보다 강력한 시각적 증거 (없으면 로고/지도 이미지 대체) |
| **Project Name**  | OO군 어촌뉴딜 300 기본계획             | O         | 사업의 정체성. 가장 크게 표시.                               |
| **Location**      | 전라북도 부안군                        | O         | 지역성 확인.                                                 |
| **Year/Duration** | 2021 - 2023                            | O         | 사업 수행 시기 및 **완료 여부** 확인.                        |
| **Category**      | 어촌 / 기본계획                        | O         | 필터링 기준 확인.                                            |
| **Key Result**    | "방문객 200% 증가, 특산물 판매장 구축" | O         | **검증의 핵심.** 단순 수행 사실을 넘어 '성과'를 증명.        |
| **Tags**          | #하드웨어 #주민교육                    | X         | 카드의 복잡도를 줄이기 위해 숨길 수 있음.                    |

## 📍 공무원 시나리오 역설계 검증

**Scenario:** 전라북도 OO군 해양수산과 주무관이 "우리 군의 어촌뉴딜 사업과 유사한 성공 사례"를 찾고 싶음.

1.  **진입 (0초):** Portfolio 페이지 접속. 전국에 꽉 찬 핀을 보고 "오, 전국구 꽤 많이 했네" 인식.
2.  **조작 (15초):**
    - 필터에서 `사업유형: 어촌` 선택 -> 내륙(농촌) 핀 사라짐.
    - (Option) 지도에서 `전라북도` 지역 줌인 또는 필터에서 `전라` 선택.
3.  **탐색 (30초):**
    - 남아있는 핀 중 바닷가 근처 핀 하나를 클릭 or 우측 리스트 스크닝.
    - "부안군 OO항 사례" 카드 발견.
    - 카드 내용 확인: "준공 후 주민 소득 1.5배 증대" (Key Result) 문구에 꽂힘.
4.  **확신 (50초):** 카드 클릭 -> 상세 페이지(Case Study)로 이동.

**검증 결과:**

- 클릭 수: 필터 2회 + 핀/카드 1회 = **총 3회 클릭** 내 도달 가능.
- 망설임 지점: 사업 유형 분류가 애매하면 망설일 수 있음 -> 명확한 4개 카테고리로 해결.
- 용어 이해: 주무관이 쓰는 '어촌뉴딜', '기본계획' 등 공식 용어를 그대로 사용해 위화감 없음.

## ✅ 완료 기준 체크

1.  실적이 '나열'이 아니라 '검증'으로 보이는가? -> **YES. 필터와 지도로 "원하는 조건의 수행 경험"을 능동적으로 검증하게 함.**
2.  발주처가 이 페이지에서 오래 머물 이유가 있는가? -> **YES. 단순 리스트보다 지도 탐색이 재미 요소가 있고, 유사 사례 비교를 위해 여러 핀을 눌러볼 것임.**
3.  경쟁사와 다른가? -> **YES. 대부분의 경쟁사는 텍스트 게시판 형태임. 지도 기반 UX는 압도적인 차별점.**
4.  데이터 확장 가능한가? -> **YES. 핀은 무한히 추가 가능하며, 클러스터링(Grouping) 기술로 1000개 이상도 처리 가능한 UX 구조임.**
