# GNB Planning 웹사이트 - 공공 발주 대응용 웹 활용 전략 (v1.0)

본 문서는 GNB Planning 웹사이트가 단순한 회사 홍보물이 아니라, **공공 입찰(Procurement) 과정에서 '신뢰 검증 도구(Verification Tool)'**로 작동하도록 돕는 전략 가이드입니다.

---

## 📄 B. 발주처 역할별 UX 시나리오

### 👤 시나리오 A: 실무 주무관 (8급/7급)

- **Needs:** "일이 되게 만드는 게 중요함. 사고 안 칠 업체 선호."
- **Web Usage:**
  1.  `Portfolio` 접속 -> `지역: 전남` 필터 선택.
  2.  지도에 핀 5개 확인 -> "아, 여기 전남 쪽 일 많이 해봤네." (안심)
  3.  가장 최근 사업 클릭 -> `Key Result` 확인 -> 과장님 보고용 근거 확보.

### 👤 시나리오 B: 팀장/과장 (5급/6급)

- **Needs:** "이 사업이 군수님/시장님 공약 사항인데, 성과가 확실히 나와야 함."
- **Web Usage:**
  1.  제안서의 `QR코드`로 대표 사업(Best Practice) 접속.
  2.  `Before & After` 사진 확인 -> "우리 읍내도 이렇게 바뀔 수 있겠군."
  3.  `GNB's Role` 확인 -> "단순 시설 공사가 아니라 주민 역량강화까지 했네." (신뢰)

### 👤 시나리오 C: 외부 평가위원 (교수/기술사)

- **Needs:** "변별력이 필요함. 논리적이고 전문적인가?"
- **Web Usage:**
  1.  `Insight` 메뉴 접속.
  2.  최근 발행된 '어촌뉴딜 3.0 정책 방향' 칼럼 제목 확인.
  3.  "단순 용역사가 아니라 정책 흐름을 읽고 있네." -> 정성 평가 가산점 요인.

## 📄 C. 제안서 – 웹 연계 운영 가이드

### 1. 연계 원칙 (Linking Rules)

- **No New Claims:** 제안서에 없는 내용을 웹에서 뜬금없이 주장하지 않는다. 웹은 제안서의 '증거(Evidence)' 저장소다.
- **Deep Link Only:** 메인페이지(Home) 링크는 지양한다. 평가위원이 바로 증거를 볼 수 있도록 **'특정 프로젝트 상세 페이지 URL'**을 사용한다.
- **Evidence Backup:** 제안서 제출 시점의 웹 페이지 상태를 PDF로 백업해둔다. (평가 기간 중 사이트 수정 금지)

### 2. 추천 페이지 세트 (Essential Links)

다음 5가지를 'GNB 디지털 팩'으로 정의하여 제안 시 활용한다.

1.  **[Map View]:** 전국 수행 현황 (규모 증명)
2.  **[Signature Project]:** 가장 유사한 성공 사례 (역량 증명)
3.  **[Process Guide]:** GNB 표준 수행 절차 (시스템 증명)
4.  **[People]:** 투입 예정 PM의 포트폴리오 (인력 증명)
5.  **[Insight]:** 관련 주제 연구 실적 (전문성 증명)

## 📄 D. 공공 발주 리스크 관리 체크리스트

### ☑️ 제안 시즌 전 필수 점검 (Pre-Check)

1.  **[실적 현행화]** 최근 6개월 내 종료된 사업이 '완료(Completed)' 상태로 업데이트되었는가?
    - _Risk:_ 심사위원이 "이거 아직도 진행 중인가요?"라고 물으면 감점.
2.  **[보안 준수]** 제안서에 '비공개'로 명시한 사업이 웹사이트에 노출되어 있지 않은가?
    - _Risk:_ 정보 보안 위반으로 입찰 자격 제한 가능성.
3.  **[경쟁사 대비]** 경쟁사가 최근 수주한 사업과 겹치는 실적이 'GNB만의 성과'로 잘 포장되어 있는가?
    - _Strategic:_ 경쟁사가 '참여'만 한 사업을 우리가 '주관'했다면 그 차이를 `Role` 필드에 명확히 기재.
4.  **[링크 유효성]** 제안서에 박아둔 QR코드/URL이 404 에러 없이 작동하는가?

## ✅ 완료 기준 (Self Check)

1.  **보완성:** 웹사이트가 제안서의 '지면 부족' 한계를 극복해주는가? **(YES)** -> 사진과 데이터를 무제한 수록 가능.
2.  **설명력:** 주무관이 상급자에게 "여기 웹사이트 한번 보세요"라고 자신 있게 말할 수 있는가? **(YES)** -> 정리된 Map과 Result가 있음.
3.  **검증 속도:** 평가위원이 3번의 클릭 안에 원하는 증거를 찾을 수 있는가? **(YES)** -> 필터링과 상세 연결 구조.
4.  **실무 적합성:** 제안팀이 실제로 링크를 걸 만한 가치가 있는가? **(YES)** -> 단순 홍보가 아닌 'Data Archive'니까.
